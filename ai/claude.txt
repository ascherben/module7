Citation brackets — [1], [2], etc. appear frequently and are usually noise for NLP/ML tasks. Strip with a simple regex: regexp.MustCompile([\d+]).ReplaceAllString(..., "").

Unicode normalization — non-breaking spaces (\u00a0), soft hyphens, and similar characters sometimes slip through strings.Fields because they aren't standard ASCII spaces.
If this scraper feeds an LLM or embedding pipeline, stripping [1] citation markers is the most impactful single addition. If it's just for human reading or search, what you have is probably fine.

Yes, that's a solid improvement. One thing worth noting: compiling the regex inside clean means it gets recompiled on every paragraph. Since clean is called in a hot loop, you should move the compiled regex to a package-level variable:


var citationRe = regexp.MustCompile(`\[\d+\]`)

func clean(content string) string {
    content = citationRe.ReplaceAllString(content, "")
    return strings.Join(strings.Fields(content), " ")
}
regexp.MustCompile is safe to use at package level and the compiled *Regexp is safe for concurrent use, so this works cleanly with your async scraper. Want me to make that change?
